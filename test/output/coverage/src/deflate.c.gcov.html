      37             : /*
      38             :  *  ALGORITHM
      39             :  *
      40             :  *      The "deflation" process depends on being able to identify portions
      41             :  *      of the input text which are identical to earlier input (within a
      42             :  *      sliding window trailing behind the input currently being processed).
      43             :  *
      44             :  *      The most straightforward technique turns out to be the fastest for
      45             :  *      most input files: try all possible matches and select the longest.
      46             :  *      The key feature of this algorithm is that insertions into the string
      47             :  *      dictionary are very simple and thus fast, and deletions are avoided
      48             :  *      completely. Insertions are performed at each input character, whereas
      49             :  *      string matches are performed only when the previous match ends. So it
      50             :  *      is preferable to spend more time in matches to allow very fast string
      51             :  *      insertions and avoid deletions. The matching algorithm for small
      52             :  *      strings is inspired from that of Rabin & Karp. A brute force approach
      53             :  *      is used to find longer strings when a small match has been found.
      54             :  *      A similar algorithm is used in comic (by Jan-Mark Wams) and freeze
      55             :  *      (by Leonid Broukhis).
      56             :  *         A previous version of this file used a more sophisticated algorithm
      57             :  *      (by Fiala and Greene) which is guaranteed to run in linear amortized
      58             :  *      time, but has a larger average cost, uses more memory and is patented.
      59             :  *      However the F&G algorithm may be faster for some highly redundant
      60             :  *      files if the parameter max_chain_length (described below) is too large.
      61             :  *
      62             :  *  ACKNOWLEDGEMENTS
      63             :  *
      64             :  *      The idea of lazy evaluation of matches is due to Jan-Mark Wams, and
      65             :  *      I found it in 'freeze' written by Leonid Broukhis.
      66             :  *      Thanks to many people for bug reports and testing.
      67             :  *
      68             :  *  REFERENCES
      69             :  *
      70             :  *      Deutsch, L.P.,"DEFLATE Compressed Data Format Specification".
      71             :  *      Available in http://tools.ietf.org/html/rfc1951
      72             :  *
      73             :  *      A description of the Rabin and Karp algorithm is given in the book
      74             :  *         "Algorithms" by R. Sedgewick, Addison-Wesley, p252.
